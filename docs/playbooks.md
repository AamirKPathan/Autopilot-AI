# Playbooks

A playbook is a reusable workflow that came from a successful supervised task.

## What to capture

- Trigger or task name
- Inputs required
- Tools used
- Approval points
- Failure modes
- Outputs and artifacts
- Recommended retry strategy

## Current rule

Do not promote a task into a playbook until Hermes has reviewed the run and the workflow has been used successfully at least once under supervision.
